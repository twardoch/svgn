In `./scripts` folder write `build_macos.sh`, `build_linux.sh` and `build_windows.cmd` (or whatever extension), which will, if ran on the given platform, build into `./dist/macos` (a universal binary, also a .pkg that installs into /usr/local/bin and also a .dmg), `.dist/linux` (a binary, and also a .tgz), and `./dist/windows` (an .exe, and a .zip). Then adapt the `./.github/workflows/release.yml` to use these scripts so that the github action only has minimal logic and most logic is in the scripts/
